简单说，`modelValue` 是“单向传值”，只负责把数据显示到开关上；`v-model` 是“双向绑定”，既负责显示数据，还能自动同步开关的状态变化到变量里——核心区别就是“能不能自动同步变化”。

### 1. `:modelValue="row.status"`：单向传值，只“显”不“更”

你可以把它理解成“给开关贴一个当前状态标签”，只做一件事：**把 `row.status` 的值（比如 1 或 0）传给开关，让开关显示对应状态**（1 是开，0 是关）。

但它有个特点：**开关被点击切换时，不会自动修改 `row.status` 的值**。  
比如你点击开关从“开”变“关”，开关的显示会变，但 `row.status` 还是原来的 1，不会自动变成 0。  
所以你必须配合 `@change` 事件（就像你代码里的 `handleStatusChange`），在事件里手动修改 `row.status`：

```javascript
// 必须手动写代码更新 row.status
const handleStatusChange = (newStatus, row) => {
  row.status = newStatus; // 开关变了，手动把新状态赋给 row.status
  // 再调用接口提交修改...
};
```

这种写法常用在“表格里的开关”（比如你代码里的 `row.status`，每一行的开关对应自己的状态），因为表格行数据是循环出来的，需要手动控制每一行的状态更新。

### 2. `v-model="form.status"`：双向绑定，又“显”又“更”

`v-model` 是个“智能同步工具”，它干了两件事：  
① 把 `form.status` 的值传给开关，让开关显示正确状态（和 `modelValue` 一样）；  
② 开关被点击切换时，**自动把新状态同步给 `form.status`**，不用你写额外代码。

比如 `form.status` 初始是 1（开），你点击开关变关：

- 开关显示会变成关；
- `form.status` 会自动从 1 变成 0，不用你手动赋值。

这种写法常用在“表单里的开关”（比如新增/修改弹框里的开关），因为表单数据是单独的，开关变化后需要自动更新表单变量，方便后续提交。

### 核心区别总结

| 特性               | `:modelValue="row.status"`       | `v-model="form.status"`            |
| ------------------ | -------------------------------- | ---------------------------------- |
| 作用               | 单向传值，只负责“显示”           | 双向绑定，既“显示”又“自动同步变化” |
| 开关变了会改变量吗 | 不会，必须用 `@change` 手动改    | 会，自动把新状态赋给变量           |
| 常用场景           | 表格行里的开关（一行一个状态）   | 表单里的开关（单独表单数据）       |
| 需不需要写额外代码 | 需要（`@change` 里手动更新变量） | 不需要（自动同步）                 |

简单记：表格里的开关用 `:modelValue + @change`（手动控），表单里的开关用 `v-model`（自动同步），这样写既简洁又不容易出问题。
